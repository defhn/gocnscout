#### How to secure the API secret key in next.js server environments?

Securing your API secret keys is paramount to maintaining data integrity and preventing unauthorized access. In Next.js server environments, we strongly recommend storing your API secret key as an environment variable (e.g., `process.env.GOCNSCOUT_API_KEY`). Crucially, all API calls involving the secret key should be made from your server-side code (e.g., within `getServerSideProps` or API routes) and never exposed directly to the client-side browser. This prevents the key from being accessible in the browser's source code, significantly enhancing security.
