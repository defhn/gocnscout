# GoCNScout × BamwoodChina：X 60 天发布日程

> 完整的 180 条英文成稿见 [`x-60-day-ready-to-post-content.md`](./x-60-day-ready-to-post-content.md)。

## 1. 目标与基本假设

这份日程基于两个网站当前已经具备的数据和产品能力：

- **GoCNScout**：20,000+ 中国出口展商/供应商公开资料，覆盖 54 个行业、城市、产品关键词、企业类型、官网和展会历史等研究字段。
- **BamwoodChina**：14,141 条竹、木、草、藤、柳、苇及相关植物基产品出口企业记录，覆盖 39 个直属海关，核心字段是产品种类、企业名称和海关注册号。
- **共同边界**：两个网站都是买方侧研究工具，不是官方认证平台，不代表验厂结果，不保证供应商质量，也不应公开个人邮箱、电话或联系人。

60 天的主要目标：

1. 建立一个具有辨识度的英文 `China sourcing intelligence` 账号。
2. 验证哪些内容能带来目标关注者、网站注册、采购询盘和付费意向。
3. 用 BamwoodChina 测试细分类数据包和定制 shortlist 的快速变现。
4. 用 GoCNScout 积累更广泛的供应商研究受众，为订阅、报告和 shortlist 建立长期入口。

## 4. 应该使用哪种内容形态

### 4.1 短帖（占主发布量约 60%）

适合：

- 一个明确结论。
- 一个风险提醒。
- 一个数据点及其含义。
- 观点、投票和问题。
- 从线程中拆出的独立知识点。

建议长度：1–5 个短段落，一条帖只表达一个中心观点。并非所有短帖都必须带链接。

### 4.2 线程（占主发布量约 25%）

适合：

- 采购教程和检查清单。
- 城市/产业带拆解。
- 多步骤供应商研究流程。
- 排行榜和数据分析。
- GoCNScout 与 BamwoodChina 的数据库建设幕后。

建议结构：

1. 第 1 条：反常识结论或清晰收益承诺。
2. 第 2–6 条：每条只讲一个步骤或发现。
3. 倒数第 2 条：说明限制条件和买方下一步核验事项。
4. 最后一条：总结和单一 CTA。

线程通常控制在 5–8 条，不为了长度拆成十几条空洞内容。

### 4.3 数据图/信息图（占主发布量约 10%）

适合：

- 城市、行业或直属海关排行榜。
- 产品类别分布。
- 供应商研究流程。
- 工厂与贸易公司信号对照。

每张图必须注明数据范围、统计口径和日期。图中不要把“展商”“出口企业”直接写成“已验证工厂”。

### 4.4 X 长篇文章（占主发布量约 5%）

前 30 天不作为重点。第 31 天后，只有当某个线程明显表现优于平均值时，才扩写成长篇文章。

适合：

- 完整的中国供应商研究方法。
- 深度产业带报告。
- 海关注册号能证明什么、不能证明什么。
- 清洗 34,000+ 企业记录后的系统性发现。

长文的作用是建立权威和承接高意向读者，不是取代日常短帖。

## 6. CTA 规则

- 约 70% 的帖子不放外链，提供完整原生价值。
- 约 20% 使用软 CTA，例如让用户查看 Bio、回复产品名或关注下一期。
- 约 10% 使用直接 CTA，导向具体城市、产品、数据库、定价或 shortlist 页面。
- 同一条帖子只保留一个主要 CTA。
- 所有外链添加 UTM 参数，至少区分网站、内容系列和日期。

示例：

```text
?utm_source=x&utm_medium=organic_social&utm_campaign=cluster_series&utm_content=day_08_yongkang
```

## 9. 内容制作工作流

每周集中用 3–4 小时完成：

1. 从数据库导出当周选题需要的聚合数字。
2. 人工检查统计口径和异常值。
3. 写 2 条线程、7–10 条短帖、2 个互动问题。
4. 制作 1–2 张数据图。
5. 为需要导流的帖子生成 UTM 链接。
6. 提前排期核心内容，但保留约 25% 空位用于实时互动。
7. 发布前检查文案是否错误使用 `verified factory`、`official database` 或 `guaranteed` 等表述。

## 10. 60 天指标与复盘

### 每周记录

- 每条帖子的曝光、点赞、回复、转发和书签。
- Profile visits 和新增关注。
- 外链点击及其 UTM 来源。
- GoCNScout 免费注册数。
- BamwoodChina 免费预览、询盘和数据包意向。
- DM 中出现的具体产品、采购阶段和预算信号。

### 核心诊断逻辑

| 现象 | 可能问题 | 下一步 |
|---|---|---|
| 有曝光、无 Profile visit | 内容有娱乐性但定位弱 | 强化专业身份和系列名称 |
| 有 Profile visit、无点击 | Bio 和置顶帖 CTA 不清楚 | 明确两个网站分别解决什么问题 |
| 有点击、无注册/询盘 | 落地页价值或信任不足 | 优化首屏、免费价值和字段说明 |
| 有注册、无付费 | 定价、字段深度或交付价值不足 | 测试小类目包和人工 shortlist |
| Bamwood 内容询问多 | 垂直需求得到验证 | 增加细分类数据包和案例 |
| GoCNScout 数据图传播好 | 长期媒体定位得到验证 | 增加城市 × 行业原创研究 |

### 第 30 天决策

- 保留表现最好的 3 个内容系列。
- 停止连续 4 次低于账号平均表现且没有商业信号的系列。
- 选择一个 BamwoodChina 小类目做付费测试。
- 选择一个 GoCNScout 行业做连续专题。

### 第 60 天决策

- 如果 BamwoodChina 带来具体采购需求，继续以细分类数据包和 shortlist 为主。
- 如果 GoCNScout 带来更多注册和跨行业需求，将其提升为内容主线。
- 如果两个产品都有人询问，继续一个主账号、两个产品承接，不拆分账号。
- 如果只有泛曝光没有商业行为，优先修改受众、CTA 和落地页，不要简单增加发帖数量。

## 11. 发布前合规检查

每条内容发布前确认：

- 没有把展商、出口企业或有海关注册号的企业称为“已验证工厂”。
- 没有承诺质量、产能、认证或当前经营状态。
- 没有公开个人联系方式和敏感字段。
- 数字来自实际数据库查询，并标明统计范围。
- 精确比例、成本节省和数据衰减率有可靠依据；否则使用定性表达。
- Supplier Spotlight 明确是研究案例，不是推荐或背书。
- 自动化只用于草稿、统计、图表和排期，最终内容由人工审核。
- 不批量发布重复文案，不自动关注、自动点赞或向陌生用户群发回复与私信。
